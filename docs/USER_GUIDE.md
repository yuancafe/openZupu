## 1. 系统基础概念

在使用系统前，了解以下几个简单的词汇能帮您快速理解系统结构：
*   **项目（Project）**：相当于**一本独立的谱书**（如《李氏族谱·华南支脉》）。您可以创建多个项目来管理不同的支派。
*   **成员/族人（Person）**：家谱中的每一个个体。
*   **亲属关系（Kinship Relation）**：连接族人之间的纽带，如父子、母子、配偶等。
*   **文献来源（Source）**：您录入信息的依据，如某本纸质老谱书、某块墓碑、或是宗祠里的石刻。
*   **修谱版本（Revision）**：每一次对家谱的修改都会被系统记录下来，方便以后查证“是谁在什么时候改了什么”。

## 2. 日常使用：新建谱书与录入准备

### 第一步：注册与登录
1. 打开浏览器，访问系统网址（通常由宗亲会的系统管理员提供）。
2. 点击 **“注册”** 按钮，填写您的用户名、电子邮箱和密码。
   > [!IMPORTANT]
   > 请妥善保管您的密码，不要泄露给他人，以防家族数据被恶意篡改。
3. 注册成功后，输入账号密码登录系统。

### 第二步：创建一本“新谱书”
1. 登录后，在主界面点击 **“创建新项目”**。
2. 输入项目名称（例如：`张氏大成宗谱_宣城支派`）和描述。
3. 点击提交，即可进入这本谱书的管理后台。

## 3. 核心操作：录入族人与建立世系

### 录入族人基本信息
在项目详情页中，点击 **“添加族人”**，您可以填写以下内容：
1.  **姓名**：系统将姓（如“陈”）和名（如“友谅”）分开录入，便于系统自动按姓氏归类。
2.  **字辈/排行**：如果您有字辈（如“宗”、“传”、“绍”），可在描述中记录，方便同名族人区分。
3.  **性别**：选择男或女。
4.  **生卒日期（重要：支持传统纪年）**：
    *   除了普通的公历日期（如 `1920-05-12`），系统还支持录入**传统日期描述**（例如：`清光绪三十年甲辰冬月初五`）。
    *   您可以勾选 **“是否健在”**。若取消勾选，系统会自动显示该长辈已故。
5.  **常住地/祖籍**：记录族人的出生地、迁徙地和安葬地。
6.  **🖼️ 生平照片/头像上传 (V1.4)**：
    *   在族人详情页左上角，点击 **“上传照片”** 按钮。
    *   选择本地照片文件，系统会自动读取并以高兼容性的 Base64 数据编码安全保存在数据库中。
    *   上传后，照片会实时呈现在当前卡片、后代世系树和祖先世系树节点中。如果需要删除照片，点击下方的 **“删除”** 按钮即可。

### 建立“父子/配偶”世系链
谱系关系是家谱的核心，OpenZupu 提供了极其简单的亲属关系建立方法：
1.  进入 **“亲属关系管理”** 页面，点击 **“添加关系”**。
2.  **选择起点和终点**：例如，从“张建国”到“张小明”。
3.  **选择关系类型**：
    *   `BIOLOGICAL_FATHER_OF`（亲生父亲）
    *   `BIOLOGICAL_MOTHER_OF`（亲生母亲）
    *   `SPOUSE_OF`（配偶/夫妻）
    *   `ADOPTIVE_FATHER_OF` / `ADOPTIVE_MOTHER_OF`（养父/养母）
4.  点击保存。系统将自动生成家族血缘网络，并能据此自动推导兄弟姐妹等关系。

## 4. 高级功能：文献管理与文献识别

### 文献来源与学术考证 (Citations & Sources) (V1.7)
家谱讲究“字字有据”，尤其对于远祖的生卒年、名讳等，各家老谱记载可能存在冲突。OpenZupu 引入了学术级的文献考证与佐证引用系统：
1.  **登记文献来源 (Sources Console)**：
    *   在项目详情页中，点击 **“文献来源”** 选项卡。
    *   点击 **“登记文献”**，可录入各种参考文献，包括：纸质宗谱/族谱、墓志铭/碑刻、历史档案、手稿等。
    *   支持详细记录文献的朝代（例：明代）、作者/修撰者、年份、卷册、页码以及网页链接等。
2.  **为信息附加佐证 (Person Citations)**：
    *   进入族人详情页，在最下方的 **“文献考证与佐证”** 卡片中，点击 **“添加考证”**。
    *   **选择考证字段**（如：字、谱名、出生日期、生平传记等）、选择已登记的文献来源，并录入**原始文献引用 (Verbatim Excerpt)** 和您对该文字的**现代考证阐释 (Commentary)**。
    *   **设定可信度 (Confidence Level)**：支持设置 `HIGH`（极高/一手史料）、`MEDIUM`（中等/侧面史料）、`LOW`（偏低/推测疑点），建立严谨的学术评级。
3.  **交互式上标引文标号 (Interactive Footnote Badges)**：
    *   一旦为某字段附加了佐证，该字段在表单中的标签旁会自动出现一个小巧的书卷超链接上标（如 `[1]`）。
    *   点击该上标将以精美的平滑动画滚动并高亮指向页面最下方的对应考证行，方便查阅人立刻核实原文。

### 纸质谱书文字识别 (OCR)
如果您手头有大量旧纸质谱书的照片，手动打字太慢，可以使用 **“文字识别任务”** 功能：
1.  点击 **“新建识别任务”**，上传谱书单页的照片。
2.  系统后台将自动识别图片中的繁简中文字符，并将其转换为电子文本。
3.  识别完成后，您可以复制文本，直接粘贴到族人信息或事迹介绍中，省去繁重的打字工作。

## 5. 数据整理：图谱生成与数据导出

### 自动生成“五世图”（世系树）
1.  在项目页面，点击 **“世系图”** 或 **“树状查看”**。
2.  输入您想要作为“根节点”的先祖名字（例如：始祖“周德兴”）。
3.  系统将自动向下检索 **五代（或自定义代数）** 内的所有子孙，生成一张清晰的嵌套树状图，直观展示世系传承。
4.  **🪭 世系扇形图 (Fan Chart) (V1.6)**：
    *   在子选项卡中选择 **“世系扇形图”**，系统会以选定的根祖先为中心绘制一张精美的圆环扇形图。
    *   中心为祖先，自里向外呈放射状扇区展示第二代（子女）、第三代（孙辈）及第四代（曾孙辈），每个扇形区块均支持 hover 高亮及点击，方便大族谱整体代际规模的查看。

### 🏮 中式竖排世系排版与打印 (V1.5)
为了满足宗亲会排版印刷老族谱、挂图印制的需求，系统提供了符合国家级档案标准的竖排右起（从右向左）排版：
1.  在“世系图谱”选项卡中，选择 **“苏式世系图”** 或 **“欧式世系记”**。
2.  **🏮 苏式世系图**：模仿经典“行序版”，名字和生年竖向书写，世代从右至左展开。同一世房族并排显示，并带有红色连接线指引支系脉络。
3.  **📜 欧式世系记**：模仿经典“房志版”，为过滤出的宗亲列表绘制分栏卡片。每一栏文字由右向左、自上而下书写，全面记载该人的名讳、生卒年、娶妻、生子、功名以及详细行状，极具古籍线装书的排版美感。
4.  **⚙️ 控制台个性化筛选**：
    *   **世支房祖选择**：可以选择某位祖先作为该房支的起点，只排版其名下分支，避免整个数据库混在一起。
    *   **世系范围**：可输入起始与截止世代（例如只打 `第12世` 到 `第15世`）。
    *   **隐私保护**：勾选“过滤在世活人”，所有健在族人的详细传记将隐藏，显示为“在世”，保护现代人隐私安全。
5.  **🖨️ 一键打印/导出PDF**：
    *   调整好筛选范围后，点击 **“排版与打印”**。
    *   系统会自动触发浏览器的打印预览，并利用样式表自动隐藏侧边菜单、页眉、页脚等无关元素，自动适配 **A4横向纸张宽度**，供您直接连上打印机印制或另存为 PDF 文件。

### 📊 数据统计看板与地缘迁徙时空轴 (V1.6)
OpenZupu 整合了业界一流宗谱管理系统的统计与时空追踪特性，提供两个独立的高级分析面板：
1.  **📊 数据统计 (Statistics)**：
    *   点击“数据统计”选项卡，即可进入系统的数据分析看板。
    *   看板自动为您计算：当前谱籍总人数、DNA 样本覆盖度、族人平均寿命（基于已故先人精确出生与过世年份计算）、以及世代代际跨度。
    *   下方提供了男女比率进度条、各世代族人分布柱状图，以及 Y-DNA / mtDNA 基因单倍群分布总表，帮助宗亲会对家族整体数据覆盖度与健康度一目了然。
2.  **🗺️ 迁徙历史 (Migration)**：
    *   点击“迁徙历史”选项卡，系统将遍历整本族谱中所有人物的“出生地”、“常住地”和“安葬地”。
    *   系统自动过滤并按历史年份（公元纪年）进行全局正序排序，用红褐色墨点与垂线拼接出一张横跨数百年的“宗族时空迁徙轴”。
    *   点击迁徙轴中的任何名字可直接跳转到其详情页，直观掌握先人从何地迁入、分支在何时散落各地的地缘变迁轨迹。

### 重名族人自动检测 (去重功能)
在长期修谱中，经常会出现“同名同姓”录入重复的情况。
*   点击 **“重名排查”** 按钮。
*   系统会扫描整本谱书，找出所有名字相同的人，列出来供您比对（例如：同名同姓但生卒年不同的两人，或者是误录入两次的同一个人）。

### 数据的导出与备份
为了确保数据永久不丢失，并且能与其他修谱软件兼容，OpenZupu 支持多种导出格式：
1.  **GEDCOM 格式导出（扩展名为 `.ged`）**：
    *   **什么是 GEDCOM？** 这是国际通用的家谱数据格式。
    *   导出的 `.ged` 文件可以无缝导入到其他家谱软件（如 PAF, Legacy, MyHeritage）中，方便数据交换。
2.  **Excel/CSV 表格导出**：
    *   您可以将所有人或所有亲属关系导出为标准的 Excel 表格，方便宗亲会成员在电脑上用 Excel 软件进行核对、排版和打印。
    *   > [!TIP]
    防注入保护：导出的表格经过了防公式篡改处理，并针对 Windows 系统的 Excel 软件进行了中文乱码修复，可以直接双击打开。

## 6. 常见问题与实用技巧 (FAQ)

#### Q1：族人出生日期不详，只有“大约在清朝道光年间”该怎么录入？
*   **答**：在生卒年份中，您可以不写精确的公历日期，而在“日期描述”或“事迹备注”中直接填写原始记录文字，系统能够很好地保存这些文本。

#### Q2：家谱里有“出嗣”（过继给叔伯）和“承嗣”的关系，该如何记录？
*   **答**：在建立关系时，系统提供了 `ADOPTIVE_FATHER_OF`（养父/嗣父）选项。您可以同时为该族人建立一个“亲生父亲”关系和一个“嗣父”关系，系统在生成图谱时会自动做特殊标注。

#### Q3：导出的 Excel 文件，用微软 Excel 软件打开时为什么中文显示正常，没有乱码？
*   **答**：系统在导出 CSV 表格时，在文件开头写入了特殊的“签名标记”（UTF-8 BOM），这使得微软 Excel 软件能自动识别并以正确的编码打开，彻底告别乱码。

#### Q4：多个人同时登录修改，家谱数据会乱掉吗？
*   **答**：不会。系统为所有修改操作都建立了“版本控制”和数据库事务锁定，保证数据的完整和一致。

## 7. 技术附录：如何为宗亲会部署系统

如果您是宗亲会里懂一点电脑技术的年轻人，或者档案馆的系统管理员，可以通过以下极简步骤在宗亲会的服务器或电脑上部署本系统：

### 前置要求
确保您的电脑/服务器上已安装：
*   [Docker](https://www.docker.com/)（容器化引擎）
*   [Docker Compose](https://docs.docker.com/compose/)

### 极简部署步骤
1.  将 OpenZupu 的开源代码包下载并解压到您的服务器。
2.  在包含 `docker-compose.yml` 的目录下新建一个 `.env` 文件，并配置您的密钥和环境变量：
    ```env
    JWT_SECRET=zupu-secure-secret-key-change-me
    DATABASE_URL="file:./dev.db"
    PORT=3000
    ```
3.  在终端运行以下命令启动系统：
    ```bash
    docker-compose up -d --build
    ```
4.  系统启动后，即可在浏览器中通过 `http://服务器IP:3000` 访问使用。所有的家谱数据都保存在本地的 SQLite 数据库文件中，安全性极高，不依赖任何第三方云服务。
